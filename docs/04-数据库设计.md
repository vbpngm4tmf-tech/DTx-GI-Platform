```
[users] 1---* [patients]

```

## 二、表结构

### users（用户表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PK | 主键 |
| username | VARCHAR(100) | 用户名 |
| email | VARCHAR(255) | 邮箱 |
| role | VARCHAR(50) | 角色 |
| status | VARCHAR(20) | 状态 |
| created_at | TIMESTAMP | 创建时间 |

### patients（患者表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PK | 主键 |
| patient_code | VARCHAR(100) | 患者编号 |
| age | INTEGER | 年龄 |
| gender | VARCHAR(10) | 性别 |
| diagnosis | TEXT | 诊断 |
| treatment_plan | TEXT | 治疗方案 |
| status | VARCHAR(20) | 状态 |
| created_at | TIMESTAMP | 创建时间 |

### research_projects（研究项目表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | SERIAL PK | 主键 |
| project_name | VARCHAR(255) | 项目名称 |
| pi_name | VARCHAR(100) | PI姓名 |
| institution | VARCHAR(255) | 机构 |
| status | VARCHAR(50) | 状态 |
| start_date | DATE | 开始日期 |
| end_date | DATE | 结束日期 |

## 三、索引设计

| 表名 | 索引字段 | 索引类型 |
|------|----------|----------|
| users | email | UNIQUE |
| patients | patient_code | UNIQUE |
| research_projects | pi_name | B-Tree |

## 四、RLS 策略

所有表已启用 Row Level Security，应用角色 `dtxgi_app_user` 拥有完整读写权限。
