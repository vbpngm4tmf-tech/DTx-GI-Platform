# DTx-GI 数据平台技术架构

> **文档编号**：D03  
> **版本**：V1.0  
> **密级**：内部公开

---

## 一、架构设计原则

| 原则 | 说明 |
|------|------|
| 高可用 | 核心服务可用性>=99.9% |
| 高安全 | 等保三级认证 |
| 可扩展 | 微服务架构 |
| 易维护 | 容器化部署 |

---

## 二、总体架构

```
接入层：微信小程序 | 医生Web端 | 管理后台 | 开放API
        ↓
网关层：API Gateway（认证/限流/路由/负载均衡）
        ↓
服务层：用户服务 | 问诊服务 | 疗法服务 | AI推理 | 通知服务
        ↓
数据层：PostgreSQL | Redis | 对象存储 | 数据仓库
        ↓
基础层：Kubernetes | Docker | CI/CD | 监控告警
```

---

## 三、技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Taro 3.x + React 18 | 跨端开发 |
| 后端 | Go + Gin / Python + FastAPI | 微服务 |
| AI推理 | TensorRT + Triton | GPU加速 |
| 数据库 | PostgreSQL + Redis | 主从架构 |
| 消息队列 | RabbitMQ | 异步处理 |
| 容器编排 | Kubernetes | 自动化部署 |
| 监控 | Prometheus + Grafana | 可观测性 |

---

## 四、数据流转

```
患者端 -> API Gateway -> 业务服务 -> 数据库
                              ↓
                        AI推理服务 <- 模型仓库
                              ↓
                        数据仓库 <- ETL流水线
                              ↓
                        数据分析/科研
```

---

## 五、安全架构

1. 传输层：全链路HTTPS，TLS 1.3
2. 接口层：JWT认证 + API签名
3. 数据层：AES-256加密 + 字段级脱敏
4. 网络层：VPC隔离 + WAF防护
