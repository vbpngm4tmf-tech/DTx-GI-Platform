## 代码规范

### 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 变量 | camelCase | `patientName` |
| 常量 | UPPER_SNAKE | `MAX_PAGE_SIZE` |
| 函数 | camelCase | `getPatientList` |
| 组件 | PascalCase | `PatientCard` |
| 文件 | kebab-case | `patient-card.tsx` |

### 代码风格

- 使用 ESLint + Prettier 统一代码风格
- 最大行长度：100 字符
- 缩进：2 个空格

## Git 分支策略

```
main: 生产分支
develop: 开发分支
feature/*: 功能分支
release/*: 发布分支
hotfix/*: 热修复分支
```

## Commit Message 规范

```
<type>(<scope>): <subject>

<type>
  feat: 新功能
  fix: 修复
  docs: 文档
  style: 格式调整
  refactor: 重构
  perf: 性能优化
  test: 测试
  chore: 构建/工具
```

## PR 流程

1. 从 `develop` 分支创建 `feature/*` 分支
2. 开发完成后提交 PR 到 `develop`
3. 代码审查通过后方可合并
4. 合并后删除功能分支

## Code Review 标准

- 代码逻辑正确
- 符合代码规范
- 有必要的注释
- 包含单元测试
- 无安全漏洞
