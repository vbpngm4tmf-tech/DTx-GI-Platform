# DTx-GI 代码仓库登记规范

> **文档编号**：E03  
> **版本**：V1.0  
> **密级**：内部公开

---

## 一、命名规范

```
格式：<项目标识>-<模块类型>-<功能名称>
示例：DTx-GI-app-patient（患者端小程序）
```

### 模块类型

| 代码 | 类型 | 说明 |
|------|------|------|
| `app` | 前端应用 | 小程序、Web应用 |
| `admin` | 管理后台 | 医生端、运营端 |
| `svc` | 后端服务 | API服务、微服务 |
| `ai` | 算法模型 | 训练、推理服务 |
| `data` | 数据处理 | ETL、数据清洗 |
| `infra` | 基础设施 | DevOps、部署 |
| `lib` | 工具库 | 组件、SDK |

---

## 二、权限管理

| 角色 | 权限 | 适用人员 |
|------|------|----------|
| Owner | 完全控制权 | 技术负责人 |
| Maintainer | 推送代码、管理CI/CD | 核心开发 |
| Developer | 推送代码、创建分支 | 普通开发 |
| Reporter | 查看代码、提交Issue | 测试/产品 |

---

## 三、分支策略（Git Flow）

| 分支 | 说明 | 来源 | 合并目标 |
|------|------|------|----------|
| `main` | 生产分支 | - | - |
| `develop` | 开发分支 | main | main |
| `feature/*` | 功能分支 | develop | develop |
| `release/*` | 预发布 | develop | main+develop |
| `hotfix/*` | 热修复 | main | main+develop |

---

## 四、Commit 规范

```
<type>(<scope>): <subject>

type: feat/fix/docs/style/refactor/perf/test/chore/security
```

---

## 五、安全要求

**禁止提交**：密码、密钥、Token、数据库连接串、个人隐私信息、商业机密
